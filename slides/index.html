How agent-browser sees a page

Instead of a screenshot, the model gets this:

[ ShopLab — Developer Tools Store ] Nav: @e1 ShopLab @e2 Shop @e3 Cart (0) @e4 Blog @e5 Contact [ Hero ] Developer Tools, Shipped Fast Everything you need to build better software. [input @e6 placeholder="Search products..."] [button @e7 "Search"] [ Featured Products ] Terminal Pro License $49.00 [button @e8 "Add to Cart"] [button @e9 "Quick View"] API Monitor $29.00 [button @e10 "Add to Cart"] [button @e11 "Quick View"]

Acting on refs

The model uses stable refs to interact — no fragile CSS selectors or XPath:

# Navigate and snapshot
agent-browser open https://localhost:3000
agent-browser snapshot -i              # get interactive wireframe

# Interact by ref
agent-browser click @e4               # click "Blog" nav link
agent-browser fill @e6 "terminal"     # fill search input
agent-browser click @e7               # click Search

# Capture evidence
agent-browser screenshot --annotate   # annotated screenshot
agent-browser errors                  # read console errors
agent-browser record                  # record a repro video

The /dogfood skill

Skills are agent instructions packaged as a SKILL.md file. Claude loads them on demand.

Install

npm i -g agent-browser
npx skills add \
  vercel-labs/agent-browser \
  --skill dogfood

Use

/dogfood https://your-app.com
Works in Claude Code, Cursor, GitHub Copilot, Cline, and 15+ other agents — skills are agent-agnostic.

What's inside SKILL.md

Two parts: frontmatter (when to trigger) + body (what to do)

---
name: dogfood
description: >
  Systematically explores and QA tests a live web application.
  Opens a browser, navigates pages, fills forms, clicks buttons,
  captures screenshots and console errors, and writes a structured
  bug report with repro steps and evidence.
triggers:
  - /dogfood
---
# Dogfood QA Agent

You are a systematic QA engineer. Given a URL, you will:
1. Open a browser session with `agent-browser open {URL}`
2. Get a snapshot: `agent-browser snapshot -i`
3. Navigate every linked page. On each page:
   - Fill and submit all forms with realistic data
   - Click all interactive elements
   - Read console errors: `agent-browser errors`
   - Capture screenshots of any anomalies
4. For each issue found, retry at least once to confirm it's reproducible
5. Write a markdown bug report to ./dogfood-report.md
...

What /dogfood does in practice

  1. Opens a browser session and orients on the starting URL
  2. Maps the site structure — all nav links and primary routes
  3. Exercises every interactive element: forms, buttons, inputs
  4. Uses realistic-looking data (not "aaa", "test@test.com")
  5. Reads the console errors log after each interaction
  6. Takes annotated screenshots for anything suspicious
  7. Retries each potential bug to confirm reproducibility
  8. Writes a structured markdown report with repro steps + evidence

Native Chrome

Bug 4 — Cart total doesn't update when quantity changes

With Terminal Pro License at qty=3, the per-item subtotal correctly shows $147.00, but the order total remains stuck at $52.92 (the initial qty=1 value including 8% tax). changeQty() updates the row subtotal but never calls updateTotals(). Only a full page refresh recalculates the total. (cart.js:66)

Bonus: The 8% tax is added silently — no tax line is shown to the user until the checkout summary.

/dogfood

ISSUE-003 · Critical · functional
URL: localhost:3000/app/cart.html · Repro video attached

When increasing item quantity in the cart, the per-row subtotal updates correctly (3 × $49.00 = $147.00), but the order summary section stays frozen at the original single-unit values ($49.00 / $3.92 / $52.92). A customer would be shown $52.92 at checkout but should see $158.76.

Steps: Add Terminal Pro License → go to cart → click + twice → observe order summary unchanged.
4 screenshots · 1 repro video · confirmed reproducible.
Chrome found the source line immediately (cart.js:66) and a bonus tax UX issue. /dogfood produced a hand-off-ready report with repro evidence — no DevTools required.
